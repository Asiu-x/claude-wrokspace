# UI设计规范与验收标准

## 一、设计规范概述

### 1.1 项目目标
实现一个现代化的AI能力中心管理系统，具有清晰的信息架构、友好的用户交互和响应式设计，确保在不同设备上都能提供良好的用户体验。

### 1.2 设计原则
- **一致性**：相同功能保持一致的交互方式和视觉风格
- **简洁性**：去除不必要的装饰，突出核心内容
- **直观性**：信息层次分明，操作流程简单自然
- **响应式**：适配不同屏幕尺寸，确保在移动设备上的可用性

## 三、字体规范

### 3.1 字体族
- **主字体**：`Inter` - 通用字体，支持各种语言
- **辅助字体**：`Noto Sans SC` - 中文支持
- **等宽字体**：`JetBrains Mono` - 代码、数字、技术内容

### 3.2 字号系统
| 级别 | 移动端 | 桌面端 | 使用场景 |
|------|--------|--------|---------|
| XS | 12px | 13px | 辅助信息、标签 |
| SM | 14px | 14px | 正文、次要内容 |
| Base | 16px | 16px | 正文、普通文本 |
| LG | 18px | 18px | 小标题、重要文本 |
| XL | 20px | 20px | 卡片标题 |
| 2XL | 24px | 24px | 页面标题 |
| 3XL | - | 30px | 大标题 |
| 4XL | - | 36px | 超大标题 |

### 3.3 字重
| 字重 | 数值 | 使用场景 |
|------|------|---------|
| Light | 300 | 次要信息、辅助文本 |
| Regular | 400 | 正文、普通文本 |
| Medium | 500 | 标题、重要文本 |
| Semibold | 600 | 强调标题、按钮文字 |
| Bold | 700 | 主要标题、重要提示 |

### 3.4 行高和字间距
- **行高**：
  - 标题：`1.2` - 紧凑行高，突出标题
  - 正文：`1.6` - 宽松行高，提高可读性
  - 代码：`1.5` - 适中行高，便于阅读

- **字间距**：
  - 标题：`0.025em` - 略微加宽，提高识别度
  - 正文：`0` - 标准间距

## 五、图标系统

### 5.1 图标库选择
- **Heroicons**：简洁、统一风格的图标（推荐）
- **Lucide Icons**：现代化线性图标
- **自定义Emoji**：在某些场合使用，增加亲和力

### 5.2 图标规范
- **统一尺寸**：
  - 小图标：`16px` - 按钮、标签
  - 标准图标：`20px` - 导航栏、列表

## 六、组件样式标准

### 6.1 卡片组件

```css
.card {
  background: #ffffff;
  border: 1px solid #e5e7eb;
  border-radius: 0.5rem;
  box-shadow: 0 1px 2px 0 rgba(0, 0, 0, 0.05);
  padding: 1.25rem;
  transition: all 0.2s ease;
}

.card:hover {
  border-color: #3b82f6;
  box-shadow: 0 4px 6px -1px rgba(0, 0, 0, 0.1);
  transform: translateY(-2px);
}
```

**Tailwind 类名示例：**
```tsx
<div className="bg-white rounded-lg shadow-sm border border-gray-200 p-5 hover:shadow-md hover:border-blue-300 transition-all duration-200">
  {/* 卡片内容 */}
</div>
```

### 6.2 按钮组件

**主按钮（Primary）：**
```tsx
<button className="px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600 active:bg-blue-700 transition-colors duration-200 font-medium">
  主要操作
</button>
```

**次要按钮（Secondary）：**
```tsx
<button className="px-4 py-2 bg-gray-100 text-gray-700 rounded-lg hover:bg-gray-200 active:bg-gray-300 transition-colors duration-200 font-medium">
  次要操作
</button>
```

**文字按钮（Text）：**
```tsx
<button className="px-3 py-1.5 text-blue-500 hover:text-blue-600 hover:bg-blue-50 rounded-lg transition-colors duration-200 font-medium">
  文字按钮
</button>
```

**危险按钮（Danger）：**
```tsx
<button className="px-4 py-2 bg-red-500 text-white rounded-lg hover:bg-red-600 active:bg-red-700 transition-colors duration-200 font-medium">
  危险操作
</button>
```

### 6.3 表单组件

**输入框（Input）：**
```tsx
<input
  type="text"
  className="w-full px-3 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500 transition-all duration-200"
  placeholder="请输入..."
/>
```

**选择框（Select）：**
```tsx
<select className="w-full px-3 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500 transition-all duration-200 bg-white">
  <option value="">请选择</option>
</select>
```

### 6.4 标签组件

**默认标签：**
```tsx
<span className="px-2 py-1 bg-gray-100 text-gray-700 rounded text-xs font-medium">
  标签
</span>
```

**状态标签：**
```tsx

  在线
</span>

{/* 警告状态 */}

</span>

{/* 危险状态 */}
<span className="px-2 py-1 bg-red-100 text-red-700 rounded-full text-xs font-medium">
  离线
</span>
```

### 6.5 导航组件

**导航项：**

## 七、交互要求

### 7.1 Hover效果

**卡片/按钮：**
- 背景色变化
- 阴影加深
- 轻微向上位移（-2px）
- 边框颜色变化（蓝色）

**链接：**
- 文字颜色变深
- 下划线动画

**表格行：**
- 背景色变为浅灰（#f9fafb）

### 7.2 点击反馈

**按钮按压：**
- 缩放效果（0.98倍）
- 背景色加深

**成功/失败反馈：**
- 成功：淡绿色背景闪烁
- 失败：淡红色背景闪烁

### 7.3 动画过渡参数

```css
.transition-base {
  transition: all 0.2s cubic-bezier(0.4, 0, 0.2, 1);
}

.transition-slow {
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
```

**入场动画：**
- 淡入：`opacity 0.3s ease-in-out`
- 上滑：`transform 0.4s ease-out, opacity 0.4s ease-out`

### 7.4 响应式断点

| 断点 | 宽度 | 设备类型 | 布局变化 |
|------|------|---------|---------|
| sm | 640px | 手机横屏 | 两列网格 |
| md | 768px | 平板 | 三列网格，导航展开 |
| lg | 1024px | 小型桌面 | 四列网格 |
| xl | 1280px | 中型桌面 | 最大宽度1200px |
| 2xl | 1536px | 大型桌面 | 保持最大宽度 |

## 九、验收标准

### 9.1 视觉还原度

| 检查项 | 验收标准 | 权重 |

| 页面布局 | 与设计稿保持一致，信息层级清晰 | 高 |

## 十一、注意事项

### 11.1 性能优化
- 图片尺寸适当，避免过度加载
- 使用CSS动画代替JavaScript动画
- 合理使用组件懒加载
- 避免不必要的重渲染

### 11.2 可访问性
- 确保文字对比度足够高（AA级标准）
- 提供必要的ARIA标签
- 支持键盘导航（Tab键）
-  focus状态清晰可见

### 11.3 开发规范
- 组件命名使用PascalCase
- 文件/文件夹命名使用kebab-case
- 常量使用UPPER_SNAKE_CASE
- 提交信息使用规范格式
