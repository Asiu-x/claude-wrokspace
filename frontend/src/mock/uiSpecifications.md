# UI设计规范文档

## 1. 设计规范概述

### 1.1 项目目标
实现一个现代化的AI能力中心管理系统，具有清晰的信息架构、友好的用户交互和响应式设计，确保在不同设备上都能提供良好的用户体验。

### 1.2 设计原则
- **一致性**：相同功能保持一致的交互方式和视觉风格
- **简洁性**：去除不必要的装饰，突出核心内容
- **直观性**：信息层次分明，操作流程简单自然
- **响应式**：适配不同屏幕尺寸，确保在移动设备上的可用性

## 2. 颜色体系

### 2.1 主色调
- **蓝色（Primary）**：`#3b82f6` - 主要按钮、链接、重要提示
- **浅蓝色（Light Blue）**：`#eff6ff` - 背景、边框、次要元素
- **深蓝色（Dark Blue）**：`#1d4ed8` - 强调文字、高对比度元素

### 2.2 辅助色彩
- **绿色（Success）**：`#10b981` - 成功状态、在线状态、积极反馈
- **黄色（Warning）**：`#f59e0b` - 警告状态、待处理状态
- **红色（Danger）**：`#ef4444` - 错误状态、离线状态、负面反馈
- **灰色（Neutral）**：`#6b7280` - 文本、边框、背景
- **浅灰（Light Gray）**：`#f3f4f6` - 背景、边框、悬停效果

### 2.3 语义化颜色
- **状态颜色**：
  - 在线/已发布：`#10b981`
  - 测试中/审核中：`#f59e0b`
  - 离线/不活跃：`#ef4444`
  - 草稿：`#9ca3af`

### 2.4 渐变色彩
- 主渐变：`linear-gradient(135deg, #667eea 0%, #764ba2 100%)` - 英雄区域、重要提示
- 辅助渐变：`linear-gradient(135deg, #f093fb 0%, #f5576c 100%)` - 装饰元素

## 3. 字体规范

### 3.1 字体族
- **主字体**：`Inter` - 通用字体，支持各种语言
- **辅助字体**：`Noto Sans SC` - 中文支持
- **等宽字体**：`JetBrains Mono` - 代码、数字、技术内容

### 3.2 字号系统（Responsive）
```css
/* 移动设备 */
:root {
  --text-xs: 0.75rem;  /* 12px */
  --text-sm: 0.875rem; /* 14px */
  --text-base: 1rem;   /* 16px */
  --text-lg: 1.125rem; /* 18px */
  --text-xl: 1.25rem;  /* 20px */
  --text-2xl: 1.5rem;  /* 24px */

  /* 桌面设备 */
  @media (min-width: 768px) {
    --text-xs: 0.8125rem;  /* 13px */
    --text-sm: 0.875rem;   /* 14px */
    --text-base: 1rem;     /* 16px */
    --text-lg: 1.125rem;   /* 18px */
    --text-xl: 1.25rem;    /* 20px */
    --text-2xl: 1.5rem;    /* 24px */
    --text-3xl: 1.875rem;  /* 30px */
    --text-4xl: 2.25rem;   /* 36px */
  }
}
```

### 3.3 字重
- **Light** (300)：次要信息、辅助文本
- **Regular** (400)：正文、普通文本
- **Medium** (500)：标题、重要文本
- **Semibold** (600)：强调标题、按钮文字
- **Bold** (700)：主要标题、重要提示

### 3.4 行高和字间距
- **行高**：
  - 标题：`1.2` - 紧凑行高，突出标题
  - 正文：`1.6` - 宽松行高，提高可读性
  - 代码：`1.5` - 适中行高，便于阅读

- **字间距**：
  - 标题：`0.025em` - 略微加宽，提高识别度
  - 正文：`0` - 标准间距

### 4.2 组件间距
- **卡片间距**：`1.5rem` - 卡片之间的间距
- **内部间距**：`1rem` - 卡片内部内容间距
- **按钮间距**：`0.5rem` - 按钮之间的间距

### 4.3 布局间距
- **内容区域宽度**：`1200px` - 最大宽度，居中对齐
- **侧边栏宽度**：`256px` - 固定侧边栏宽度
- **内边距**：`1rem` - 页面内容与屏幕边缘间距

## 7. 交互要求

### 7.1 Hover效果
- **卡片/按钮**：
  - 背景色变化
  - 阴影加深
  - 轻微放大
  - 边框颜色变化

- **链接**：
  - 文字颜色变深
  - 下划线动画

### 7.2 点击反馈
- **按钮按压**：
  - 缩放效果（0.98倍）
  - 背景色加深

- **成功/失败反馈**：
  - 成功：淡绿色背景闪烁
  - 失败：淡红色背景闪烁

### 7.3 动画过渡参数
```css
.transition-base {
  transition: all 0.2s cubic-bezier(0.4, 0, 0.2, 1);
}

.transition-slow {
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}

.transition-bounce {
  transition: all 0.4s cubic-bezier(0.68, -0.55, 0.265, 1.55);
}

.transition-smooth {
  transition: all 0.5s cubic-bezier(0.16, 1, 0.3, 1);
}
```

### 7.4 响应式断点
```css
/* 移动设备 */
@media (min-width: 640px) {

}

@media (min-width: 768px) {
  /* 小型桌面设备 */
}

  /* 中型桌面设备 */
}

@media (min-width: 1280px) {
  /* 大型桌面设备 */
  .container {
    max-width: 1200px;
  }
}

@media (min-width: 1536px) {
  /* 超大型桌面设备 */
}
```

## 9. 参考页面结构分析

3. **内容区域**：

   - 各业务模块推荐内容
   - 最新动态

### 9.2 样式参考
- **配色**：使用蓝色系为主色调，辅以绿色、黄色、红色等语义化颜色
- **字体**：使用现代无衬线字体，层次分明

- **交互**：响应式设计，hover和点击有明确的视觉反馈

### 9.3 内容模块分析

## 10. 验收标准

### 10.1 视觉还原度
- 页面布局与参考页面保持一致
- 颜色、字体、间距符合规范
- 响应式设计在各断点表现良好

### 10.2 交互体验
- 所有交互有明确的视觉反馈
- 页面加载有适当的动画效果
- 搜索、筛选、排序功能正常工作

### 10.3 功能完整性
- 所有页面能够正常访问和导航
- 列表页面支持筛选和分页
- 详情页面展示完整信息
- Mock数据与真实API响应格式一致

### 10.4 代码质量
- 组件化开发，结构清晰
- 代码符合规范，易于维护
- 充分注释，便于理解

## 11. 注意事项

### 11.1 性能优化
- 图片尺寸适当，避免过度加载
- 使用CSS动画代替JavaScript动画
- 合理使用组件懒加载

### 11.2 可访问性
- 确保文字对比度足够高
- 提供必要的ARIA标签
- 支持键盘导航

### 11.3 浏览器兼容性
- 支持现代浏览器（Chrome、Firefox、Safari、Edge）
- 考虑IE11兼容性（如果需要）
