# Git 提交规范

本项目采用约定式提交（Conventional Commits）规范，所有提交信息必须遵循以下格式：

## 提交格式
```
<类型>(<作用域>): <主题>

<正文>

<页脚>
```

### 1. 类型（必填）
必须是以下类型之一：
| 类型 | 描述 |
|------|------|
| feat | 新功能 |
| fix | 修复bug |
| docs | 文档更新 |
| style | 代码格式调整（不影响代码逻辑） |
| refactor | 代码重构（既不新增功能也不修复bug） |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具/依赖等配置调整 |
| revert | 回滚提交 |

### 2. 作用域（可选）
说明提交影响的范围，例如：
- `components/Button`
- `pages/Dashboard`
- `api`
- `style`

### 3. 主题（必填）
- 不超过50个字符
- 使用动词开头，例如：新增、修复、调整、优化
- 结尾不加句号
- 简明扼要描述提交内容

### 4. 正文（可选）
- 详细描述提交的原因、修改内容、设计思路等
- 每行不超过72个字符

### 5. 页脚（可选）
- 关联Issue：`Closes #123`
- 破坏性变更：`BREAKING CHANGE: 描述变更内容`

## 提交示例
### 简单提交
```
feat(components): 新增用户卡片组件
```

### 带正文的提交
```
fix(api): 修复用户列表分页参数错误

- 修复page参数从0开始的问题
- 调整limit参数默认值为10
- 添加参数校验逻辑

Closes #456
```

### 破坏性变更
```
refactor(button): 重构Button组件API

BREAKING CHANGE: Button组件的type属性取值变更，原primary改为default，原default改为secondary
```

## 禁止的提交
- 空提交
- 提交信息模糊，例如：`修复bug`, `调整代码`, `更新`
- 一次提交包含多个不相关的修改
- 提交未完成的代码
