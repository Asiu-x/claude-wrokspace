# 周报生成技能

基于 Obsidian X智库中的日报记录，自动生成结构化周报，支持输出 Markdown、飞书HTML和PNG图片三种格式。

## 关键路径

- **日报目录**：`/Users/xccx/Library/Mobile Documents/com~apple~CloudDocs/工作文件/X智库/01-日记/`
- **周报目录**：`/Users/xccx/Library/Mobile Documents/com~apple~CloudDocs/工作文件/X智库/01-日记/11-周报/`
- **项目台账**：`/Users/xccx/Library/Mobile Documents/com~apple~CloudDocs/工作文件/X智库/08-项目资产/项目台账.md`
- **历史周报**：周报目录下的 `2026-{周数}.md` 文件，可作为格式参考
- **HTML转PNG脚本**：`scripts/html2png.py`

## 工作流程（严格按顺序执行）

### 阶段一：数据收集

1. 确定当前周数和日期范围（周一至周五）
2. 读取日报目录下本周所有日报文件（格式 `YYYY-MM-DD.md`）
3. 读取上一周的周报文件，了解项目延续情况和格式规范
4. **读取项目台账**（`08-项目资产/项目台账.md`），了解各项目当前阶段和历史状态，用于判断项目阶段变化
5. 如需了解周报模板结构，读取 `references/weekly-report-template.md`

### 阶段二：生成 Markdown 周报初稿

1. 将日报条目按项目维度归类整合（同一项目在不同天的工作合并描述）
2. 按"售前/交付/流程"三线分类，每个项目包含：项目阶段、本周工作、风险项
3. 根据本周推进内容推导下周计划
4. 将周报写入 `11-周报/2026-{周数}.md`
5. **必须停下来，将完整内容展示给用户审核**

### ⚠️ 审核关卡（必须执行）

生成 Markdown 初稿后，**必须使用 AskUserQuestion 工具**询问用户：

> 周报初稿已生成，请审核以下内容：
> 1. 项目的售前/交付分类是否正确？
> 2. 工作内容描述是否准确？
> 3. 是否需要增加AI提效成果专项？
> 4. 下周计划是否需要调整？
>
> 确认无误后我将生成飞书HTML版和PNG图片。

**在用户明确确认之前，禁止进入阶段二-B和阶段三。** 用户可能会要求修改项目分类、调整描述、增删内容等，需逐一处理直到用户满意。

### 阶段二-B：更新项目台账

用户确认周报内容后，根据本周周报更新 `08-项目资产/项目台账.md`：

1. **新增项目**：本周周报中出现但台账中没有的项目，状态标记为 🔵 新增
2. **更新阶段**：对比台账中的阶段与本周周报的阶段描述，如有变化则更新。售前阶段链路：
   - `需求沟通 → 方案设计 → 方案审核 → 报价确认 → 预投招标 → 签约`
   - 签约完成 → 移入"交付项目"区域
3. **更新状态**：根据本周推进情况更新状态标识
   - 🔵 新增：本周新进入的项目
   - 🟢 推进中：本周有实质性推进
   - 🟡 跟进中：等待外部响应或需持续关注
   - 🔴 风险：存在风险项
   - ⚫ 验收完结：项目验收完成（仅用户手动标记）
4. **更新进展摘要**：用本周工作内容覆盖摘要字段
5. **更新最近更新周数**
6. **流程支撑事务**：同步更新流程支撑事务的状态和摘要
7. **完结处理**：用户手动标记验收完结的项目移入"已完结项目"区域
8. 更新台账顶部的"最近更新"时间戳

> 注意：使用 Bash heredoc 写入台账文件（iCloud 路径兼容性要求）。

### 阶段三：生成飞书HTML和PNG图片

仅在用户确认 Markdown 内容后执行：

1. 读取 `references/html-template-guide.md` 了解HTML样式规范
2. 基于确认后的 Markdown 内容生成单行压缩HTML文件，保存为 `11-周报/2026-{周数}-飞书版.html`
   - 所有HTML标签必须压缩在同一行，标签之间不允许有换行符（飞书会将换行解析为空行）
   - 顶部包含数据概览卡片（售前项目数、交付项目数等）
   - 项目阶段使用彩色标签：售前=蓝色(tag-blue)、交付/签约=绿色(tag-green)、变更=橙色(tag-orange)
   - 风险项有内容时红色加粗(risk-warn)，无风险灰色(risk-none)
   - AI提效板块使用蓝色渐变卡片样式(highlight-box)
3. 使用 `scripts/html2png.py` 将HTML转为PNG图片
   ```bash
   python3 /Users/xccx/.claude/skills/weekly-report/scripts/html2png.py <html路径> <png路径> 1080 4200
   ```
4. 用 `open` 命令打开生成的PNG图片供用户查看

## 输出文件清单

每次生成周报产出3个文件 + 更新1个文件：

| 文件 | 路径 | 用途 |
|------|------|------|
| Markdown源文件 | `11-周报/2026-{周数}.md` | Obsidian存档 |
| 飞书HTML | `11-周报/2026-{周数}-飞书版.html` | 浏览器打开后复制粘贴到飞书 |
| PNG图片 | `11-周报/2026-{周数}-周报.png` | 直接发送给领导 |
| 项目台账（更新） | `08-项目资产/项目台账.md` | 项目状态总表，每周随周报更新 |

## 注意事项

- 日报中的简要条目需扩写为完整的工作描述，但不要过度发挥，保持忠于原始记录
- 项目的售前/交付归属不可自行假设，如不确定必须询问用户
- AI提效板块为可选内容，仅在用户要求时加入
- 语气要求：职业、精炼、结果导向
- 周报命名规则：周数为当年第几周（ISO周数）
